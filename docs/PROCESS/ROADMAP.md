# ROADMAP

План реализации по вертикальным слайсам, выведенный из [[REQUIREMENTS]] и приоритетов оценки
([[BUSINESS_METRICS]]). Bootstrap (доки + тест-контур) завершён; основная разработка стартует после
апрува. Каждый слайс закрывается своими тестами из [[TEST_PLAN]] и критериями из [[ACCEPTANCE_CRITERIA]].

## Слой 0 — Bootstrap (текущий, завершается этой сессией)
Документация, инварианты, ADR, семантическая память, полный POM-контур API-тестов, CI, git/push.
Код продукта не пишется.

## Слайс 1 — Каркас приложения и инфраструктура (FastAPI)
`app/` со слоями (api/services/repositories/schemas/models/core), подключение к БД, Alembic,
единый обработчик ошибок (ADR-003), конфиг из env, логирование (NFR-5/6), health-эндпоинт,
автогенерация OpenAPI (NFR-8). **DoD:** контур поднимается, конверт ошибок работает, миграции применяются.

## Слайс 2 — Module 1: Employees (15 баллов)
Сущность, CRUD, деактивация (ADR-006), уникальность email, фильтры/поиск/пагинация (FR-E*).
**DoD:** спеки `test_employees.py` зелёные; AC-E1..E9 выполнены.

## Слайс 3 — Module 2: Tickets (20 баллов)
Сущность, создание (status=new), связь с сотрудником, назначение (→in_progress), машина статусов
(ADR-005), resolved_at, статистика, фильтры/пагинация (FR-T*).
**DoD:** спеки `test_tickets.py` зелёные; AC-T1..T10 выполнены.

## Слайс 4 — Module 3: Integration & Security (25 + безопасность 10)
API-клиенты (хеш ключа, одноразовый показ, ADR-004), X-API-Key auth, rate limit (401/403/429/422),
сохранение запросов, аудит (ip/user_agent), список запросов/аудита, обработка employee_sync с
идемпотентностью (ADR-007). **DoD:** спеки `test_integration_*`, `test_audit.py` зелёные; AC-I*.

## Слайс 5 — Финализация (документация и запуск 5, БД 10)
README с примерами запросов/ответов, Postman collection (если нужно сверх OpenAPI), индексы (бриф §47),
ответы на доп. вопросы §43–52, список улучшений. **DoD:** проект сдаваем по чек-листу §6 брифа.

## Предлагаемый первый слайс после апрува
**Слайс 1 — каркас приложения** (без него остальные слайсы не зелёные). Затем — Слайс 2 (Employees),
поскольку модуль 1 — зависимость для модулей 2 и 3 (заявки ссылаются на сотрудников; employee_sync
пишет в справочник).
