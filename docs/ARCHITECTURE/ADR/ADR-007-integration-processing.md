# ADR-007: Обработка интеграционных запросов — идемпотентность и employee_sync

## Status
Accepted

## Context
Бриф §4.5 описывает обработку запроса администратором: для `employee_sync` создать/обновить
сотрудника (матч по external_id, иначе по email); успех→processed, ошибка→failed+error_message;
повторная обработка processed запрещена. Требования FR-I14..I17; инварианты INV-P11, INV-P12.
Также бриф §45 ставит вопрос о гонке двух администраторов.

## Decision
- **Жизненный цикл запроса:** accepted → (process) → processed | failed. rejected — отдельная
  ветка отказа на приёме. Статусы ∈ {accepted, rejected, processed, failed} (INV-D3).
- **Идемпотентность:** обработка запроса в статусе processed запрещена → 409 `ALREADY_PROCESSED`
  (INV-P11). Защита от гонки (§45) обеспечивается на уровне БД: атомарный переход статуса
  (условный UPDATE/`SELECT ... FOR UPDATE` или уникальный guard) — деталь реализации, зафиксирована
  как требование к продукту, проверяется тестом повторной обработки (AC-I11).
- **employee_sync:** сопоставление сотрудника по `payload.external_id`; при его отсутствии — по
  `payload.email` (INV-P12). Найден → update, иначе → create. Результат: processed + message.
- **Ошибки обработки:** невалидный payload/сбой → status=failed, текст в error_message (INV-P16/FR-I16).

> `[ASSUMPTION]`: поле `external_id` сотрудника не указано в сущности §2.2. Для устойчивого матча
> по external_id потребуется хранить external_id (например, отдельная колонка/таблица маппинга).
> Зафиксировано как OPEN QUESTION Q-8; в тест-контуре матч проверяется через наблюдаемый эффект
> (создание/обновление сотрудника с тем же email/полями).

## Consequences
### Positive
- Повторная обработка безопасна; гонка администраторов разрешается детерминированно.
- Связка модуля 3 и модуля 1 явная и тестируемая.
### Negative
- Нужна доп. колонка/маппинг для external_id (расширение модели §2.2) — см. Q-8.

## Alternatives considered
### Матч только по email
Отклонено: бриф явно требует приоритет external_id.
### Разрешить повторную обработку (перезапись)
Отклонено: нарушает INV-P11 и §4.5.
